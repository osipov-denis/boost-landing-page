# GitHub Pages Deployment Guide

## 📋 Инструкция по деплою на GitHub Pages

### Шаг 1: Создайте репозиторий на GitHub

1. Перейдите на https://github.com/new
2. Введите название репозитория: `boost-landing-pages`
3. Сделайте репозиторий **Public** (обязательно для GitHub Pages)
4. **НЕ** отмечайте "Add a README file" (у нас уже есть README.md)
5. Нажмите "Create repository"

### Шаг 2: Свяжите локальный репозиторий с GitHub

После создания репозитория, GitHub покажет вам команды. Выполните:

```bash
cd "c:\Users\soul\.cursor\SoulLab\Zamesin"
git remote add origin https://github.com/osipov-denis/boost-landing-pages.git
git branch -M main
git push -u origin main
```

### Шаг 3: Активируйте GitHub Pages

1. Перейдите в репозиторий на GitHub
2. Нажмите **Settings** (настройки)
3. В меню слева выберите **Pages**
4. В разделе **Build and deployment**:
   - **Source**: выберите "Deploy from a branch"
   - **Branch**: выберите "main" и папку "/ (root)"
   - Нажмите **Save**

### Шаг 4: Дождитесь деплоя

Через 1-2 минуты ваш сайт будет доступен по адресу:
```
https://osipov-denis.github.io/boost-landing-pages/
```

## 📁 Доступ к файлам

После деплоя каждый файл будет доступен по своему URL:

```
https://osipov-denis.github.io/boost-landing-pages/boost-minimalist.html
https://osipov-denis.github.io/boost-landing-pages/boost-brutalist.html
https://osipov-denis.github.io/boost-landing-pages/boost-cyberpunk.html
...
```

## 🔍 Проверка статуса деплоя

1. В репозитории на GitHub перейдите в **Settings** → **Pages**
2. Смотрите раздел "Deployment" - там появится статус
3. Когда появится зеленая галочка ✅ - деплой завершен
