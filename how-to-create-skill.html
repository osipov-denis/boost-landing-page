🎓 Как сделать скилл в Claude Code

Пошаговый гайд для новичка, который никогда не программировал

⏱ Время чтения: 10 минут

📋 Что такое скилл?

Скилл в Claude Code — это как "плагин" или "расширение" для ИИ-помощника. Вы даёте Claude специальные инструкции и инструменты, и он становится экспертом в конкретной задаче.

Простой пример: Скилл для проверки орфографии, который всегда находит ошибки и предлагает исправления. Или скилл для создания описаний товаров, который пишет продающие тексты.

🎯 Зачем нужен скилл?

  • Экономит время на повторяющихся задачах
  • Делает работу Claude более специализированной
  • Можно поделиться с другими людьми
  • Просто создать (даже если вы не программист!)

📝 Как создать скилл — Пошаговая инструкция

1 Определите, что должен делать ваш скилл

Придумайте одну чёткую задачу. Не пытайтесь сделать "всё сразу".

✅ Хорошие примеры:

  • "Проверять текст на ошибки пунктуации"
  • "Создавать описания для товаров маркетплейса"
  • "Переводить текст с английского на русский"

❌ Плохой пример:

  • "Помогать со всем" (слишком расплывчато)

2 Создайте папку для скилла

Скилл — это просто папка с файлами. Создайте её где угодно на компьютере.

mkdir my-first-skill
cd my-first-skill
💡 Совет: Название папки должно быть простым, без пробелов и русских букв. Используйте дефисы: my-cool-skill

3 Создайте главный файл скилла

В папке создайте текстовый файл с названием skill.md (именно так!)

# Название скилла

You are a helpful assistant specialized in [ваша задача].

## Instructions
[ваши инструкции для Claude]

## Examples
[примеры, если нужны]

4 Напишите инструкции для Claude

В файле skill.md опишите, что должен делать Claude. Пишите просто и понятно.

You are a spelling checker assistant.

Your task:
1. Check the text for spelling errors
2. Suggest corrections
3. Explain each correction

Always be friendly and helpful.
💡 Совет: Используйте списки и numbered steps (1, 2, 3...) — Claude любит структуру.

5 Протестируйте скилл

Запустите Claude Code и скажите:

Используй скилл из папки my-first-skill

Или просто откройте файл skill.md в Claude — он автоматически загрузит инструкции.

🎮 Пример скилла — "Улучшатель текста"

Вот полный пример простого скилла, который делает текст более читаемым:

# Text Improver Skill

You are a text improvement expert.

## Your Task
Make any text more readable and professional while keeping the original meaning.

## Steps
1. Read the user's text
2. Identify areas for improvement:
   - Long sentences → break into shorter ones
   - Complex words → replace with simpler ones
   - Passive voice → change to active voice
3. Rewrite the text
4. Show what you changed and why

## Rules
- Keep the original meaning
- Don't add new information
- Always explain your changes

## Example
User: "The report was written by me and it was sent to the boss."
You: "I wrote the report and sent it to the boss."
Changed "was written by me" → "I wrote" (active voice is clearer)

⚠️ Типовые ошибки и как их исправить

❌ Ошибка 1: "Claude не понимает, что делать"

Причина: Слишком расплывчатые инструкции.

Решение: Добавьте конкретные шаги. Вместо "Помогай с текстом" напишите:

Your task:
1. Check grammar
2. Check spelling
3. Suggest improvements

❌ Ошибка 2: "Скилл работает, но делает не то"

Причина: Не хватает примеров.

Решение: Добавьте раздел ## Examples с 2-3 примерами "вход → выход".

❌ Ошибка 3: "Claude добавляет лишнее"

Причина: Нет ограничений в правилах.

Решение: Добавьте раздел ## Rules с明确的 ограничениями:

## Rules
- Don't add new information
- Keep original tone
- Stay concise

❌ Ошибка 4: "Файл не открывается"

Причина: Неверное название или кодировка.

Решение: Проверьте:

  • Файл называется точно skill.md (не skill.txt)
  • Нет пробелов в названии
  • Файл сохранён в UTF-8 кодировке

❌ Ошибка 5: "Результаты непредсказуемые"

Причина: Claude слишком творчески интерпретирует инструкции.

Решение: Добавьте фразу:

Follow these instructions EXACTLY. Do not improvise.

🚀 Мини-шпаргалка: "Ничего не работает"

  • 🔍 Проверьте файл: называется ли он skill.md?
  • 📝 Проверьте инструкции: они конкретные?
  • 📋 Добавьте примеры: хотя бы 2-3
  • 🚫 Уберите лишнее: уберите_water, уточните задачу
  • ✅ Протестируйте: запустите на простом примере
  • 🔄 Итерируйте: улучшайте по результатам
  • 💾 Сохраняйте версии: копируйте файл перед изменениями
  • 📚 Смотрите примеры: изучите другие скиллы
  • 🤖 Попросите Claude: "Проверь мой skill.md и скажи, что улучшить"
  • ⚡ Начните заново: если совсем всё плохо — создайте новый файл с нуля

💡 Дополнительные советы

  • Начинайте просто — первый скилл должен занимать 10-15 строк
  • Тестируйте часто — проверяйте после каждого изменения
  • Сохраняйте версии — копируйте файл: skill-v1.md, skill-v2.md
  • Учитесь на примерах — смотрите другие скиллы в документации Claude
  • Не бойтесь ошибок — это нормально! Каждый скилл улучшается с опытом

🎓 Что дальше?

После создания первого скилла:

  1. Протестируйте на разных примерах
  2. Улучшайте по результатам
  3. Поделитесь с коллегами или друзьями
  4. Создайте второй скилл для другой задачи!

Помните: Каждый скилл делает Claude лучше для конкретной задачи. Начните с простого, и вы быстро научитесь!
